# Mini Lead Tracker 🚀

Легковажний, високопродуктивний API для збору, управління та відстеження лідів (заявок). Побудований на базі безсерверної (serverless) екосистеми **Cloudflare**.

## 🛠️ Стек технологій

- **Runtime (середовище виконання)**: [Cloudflare Workers](https://workers.cloudflare.com/)
- **База даних**: [Cloudflare D1](https://developers.cloudflare.com/d1/) (безсерверна SQL-база даних на рушії SQLite)
- **Кеш та конфігурація**: [Cloudflare KV](https://developers.cloudflare.com/kv/) (швидке сховище типу ключ-значення)
- **Мова програмування**: TypeScript

---

## ✨ Особливості та можливості

1. **Структуроване сховище (D1)**: Збереження лідів у реляційній базі даних із використанням SQL-індексів для швидкого пошуку, фільтрації та пагінації.
2. **Розумне кешування (KV)**: Швидке отримання списку лідів завдяки кешуванню у KV. Кеш автоматично інвалідується (очищується) при створенні, зміні статусу або видаленні лідів.
3. **Обмеження запитів / Rate Limiting (KV)**: Захист публічного ендпоінту створення лідів від спаму та зловживань (ліміт: 3 запити на хвилину з однієї IP-адреси).
4. **Динамічне керування сповіщеннями (KV)**: Можливість увімкнути/вимкнути симуляцію надсилання сповіщень на пошту «на льоту» без передеплою коду за допомогою прапорця `settings:lead_email_enabled`.
5. **Безпека та авторизація**: Захист адміністративних ендпоінтів за допомогою Bearer-токена в заголовках (`Authorization: Bearer <token>`).

## 🚦 Ендпоінти API

### Публічні ендпоінти

#### 📥 Створення ліда
- **Метод та URL**: `POST /leads`
- **Тіло запиту** (JSON):
  ```json
  {
    "name": "Олександр Коваленко",
    "email": "oleksandr.kovalenko@example.com",
    "company": "Квіти під Ключ", 
    "message": "Хочу замовити великий букет білих лілій з доставкою." 
  }
  ```
- **Відповідь** (201 Created):
  ```json
  {
    "id": 1,
    "name": "Олександр Коваленко",
    "email": "oleksandr.kovalenko@example.com",
    "company": "Квіти під Ключ",
    "message": "Хочу замовити великий букет білих лілій з доставкою.",
    "status": "new",
    "created_at": "2026-07-06T09:00:00.000Z"
  }
  ```

### Приватні ендпоінти (потребують авторизації)
*(Обов'язковий заголовок: `Authorization: Bearer <ваш_токен>`)*

#### 📋 Отримання списку лідів (з пагінацією та фільтрацією)
- **Метод та URL**: `GET /leads?page=1&status=new`
- **Параметри запиту (Query)**:
  - `page`: Номер сторінки (за замовчуванням `1`, повертає по 10 лідів на сторінку).
  - `status`: Фільтр за статусом (`new`, `contacted`, `closed`).
- **Відповідь** (200 OK):
  ```json
  [
    {
      "id": 1,
      "name": "Олександр Коваленко",
      "email": "oleksandr.kovalenko@example.com",
      "company": "Квіти під Ключ",
      "message": "Хочу замовити великий букет білих лілій з доставкою.",
      "status": "new",
      "created_at": "2026-07-06T09:00:00.000Z"
    }
  ]
  ```

#### 🔍 Отримання одного ліда за ID
- **Метод та URL**: `GET /leads/:id`
- **Відповідь** (200 OK):
  ```json
  {
    "id": 1,
    "name": "Олександр Коваленко",
    "email": "oleksandr.kovalenko@example.com",
    "company": "Квіти під Ключ",
    "message": "Хочу замовити великий букет білих лілій з доставкою.",
    "status": "new",
    "created_at": "2026-07-06T09:00:00.000Z"
  }
  ```

#### 🔄 Оновлення статусу ліда
- **Метод та URL**: `PATCH /leads/:id/status`
- **Тіло запиту** (JSON):
  ```json
  {
    "status": "contacted" 
  }
  ```
- **Відповідь** (200 OK):
  ```json
  {
    "id": 1,
    "name": "Олександр Коваленко",
    "email": "oleksandr.kovalenko@example.com",
    "company": "Квіти під Ключ",
    "message": "Хочу замовити великий букет білих лілій з доставкою.",
    "status": "contacted",
    "created_at": "2026-07-06T09:00:00.000Z"
  }
  ```

#### ❌ Видалення ліда
- **Метод та URL**: `DELETE /leads/:id`
- **Відповідь** (200 OK):
  ```json
  {
    "message": "Лід з ID 1 успішно видалено"
  }
  ```

#### 📊 Отримання загальної статистики
- **Метод та URL**: `GET /stats`
- **Відповідь** (200 OK):
  ```json
  {
    "total": 28,
    "new": 15,
    "contacted": 7,
    "closed": 6
  }
  ```

## ⚙️ Локальна розробка та запуск

### 1. Встановлення залежностей
Клонуйте репозиторій та встановіть необхідні npm-пакети:
```bash
git clone https://github.com/Hellstaff8822/mini-lead-tracker.git
cd mini-lead-tracker
npm install
```

### 2. Налаштування локальних змінних оточення
Створіть файл `.dev.vars` у кореневій директорії проєкту та додайте секретний токен авторизації:
```env
AUTH_TOKEN="your-secret-token"
```

### 3. Створення локальної бази даних (D1 Local)
Запустіть міграції, щоб створити таблицю та індекси локально:
```bash
npx wrangler d1 migrations apply mini_lead_tracker_db --local
```

*(Опціонально) Наповніть локальну базу тестовими даними:*
```bash
npx wrangler d1 execute mini_lead_tracker_db --local --file=./seeds.sql
```

### 4. Запуск сервера розробки
Запустіть локальний сервер розробки Wrangler:
```bash
npx wrangler dev
```
Локальний API буде доступний за адресою `http://localhost:8787`.

## 🚀 Деплой на Cloudflare (Production)

### 1. Застосування міграцій у хмарі
Запустіть міграції для створення бази даних у хмарному середовищі Cloudflare:
```bash
npx wrangler d1 migrations apply mini_lead_tracker_db --remote
```

### 2. Налаштування секретів на продакшені
Встановіть авторизаційний токен у хмарі:
```bash
npx wrangler secret put AUTH_TOKEN
```
*(Введіть значення вашого токена у запиті терміналу)*

### 3. Публікація (деплой)
Опублікуйте свій Worker у хмарі Cloudflare:
```bash
npx wrangler deploy
```
